# 任务编排：任务列表页

## 1. 一句话定义
任务列表页是“任务编排”一级功能的根页面，用于浏览所有 task、查看 requirement 摘要和当前运行状态，并进入单个任务。

---

## 2. 页面入口
进入方式：
- 启动桌面客户端后的默认页面
- 点击左侧一级菜单“任务编排”；即使当前位于任务工作流或 Round 详情等深层页面，也回到任务列表根页面
- 在任务编排内部面包屑点击“任务列表”

---

## 3. 页面结构

```text
┌──────────────────────────────────────────────────────────────┐
│ 统一 Page Header：面包屑 / 标题 / 副标题 / 刷新与占位操作         │
├──────────────────────────────────────────────────────────────┤
│ 任务列表工具条：快捷筛选 / 状态筛选 / 关键字搜索                 │
│ task id | 标题 / requirement 摘要 | 状态 | workflow | 最近 run | 操作 │
│                                                              │
│ 单击任务行 -> 右侧 Task Preview Sheet 按需滑出                 │
└──────────────────────────────────────────────────────────────┘
```

---

## 4. 顶部区域

顶部区域使用与任务工作流页、Round 详情页一致的统一 Page Header：面包屑在 Header 内第一行，标题和副标题在左，刷新与占位操作在右；内容区不再重复展示 workspace path 或说明文案。

### 4.1 面包屑
根页面只显示：

```text
任务列表
```

### 4.2 页面标题
标题：

```text
任务编排
```

副标题：

```text
管理 requirements、运行状态与 workflow 执行历史
```

### 4.3 页面操作
当前建议：
- 刷新
- 创建任务

“导入 requirements”不再作为任务列表页的独立入口；导入需求被收敛进“创建任务”弹窗的第一步。点击“创建任务”后，从右侧打开创建任务 Sheet：选择 `.txt` / `.md` 需求文件、填写任务标题/描述、选择或创建工作流并保存。创建任务 Sheet 内提供持久化工作流模板下拉，上次创建任务使用的模板不进入下拉列表，而是在下拉框旁以“上次创建使用的是：xxx”提示展示；点击模板名会自动选中该模板并隐藏提示。模板下拉顶部提供“新增模板”按钮，进入空白画布；每一行展示删除按钮，默认工作流禁用删除；用户修改节点、边或节点属性后，可选择“保存修改”覆盖当前非默认模板，也可将当前 workflow 另存为命名模板，保存成功后在工作流区显示轻量成功反馈。需求文件类错误展示在需求表单下方；模板保存、模板删除等工作流错误展示在工作流模板操作区下方，避免顶到整个创建流程顶部。

---

## 5. 表格筛选与搜索
任务列表页不展示页面级状态摘要卡，也不在首屏使用大数字 KPI 气泡。

任务状态聚合能力收敛到表格工具条中：
- 快捷筛选：全部任务 / 运行中 / 已完成。
- 状态筛选：全部状态 / 可恢复 / 失败 / 配置异常。
- 关键字搜索：支持 task id、标题、requirement 摘要、完整 requirement、最近 run 和 workflow 状态。

行为规则：
- 筛选和搜索都作用于同一个表格结果集。
- 切换筛选或修改搜索关键字时，分页回到第一页。
- 页面顶部不承载任务状态 tabs；tabs 属于表格工具条。
- 不为了展示统计数量而新增卡片区域，避免任务编排首页退化为运行态数据看板。

---

## 6. 任务列表

### 6.1 列表字段
每行 task 展示：
- task id
- task title
- requirement 摘要
- 当前状态
- workflow 校验状态
- 最近 run
- 操作按钮

示例：

```text
task-001  Tauri 桌面端重写  摘要...  Completed  Valid  run-003  进入
task-002  修复 provider 输出  摘要...  Resumable  Valid  run-002  进入
task-003  优化文档结构        摘要...  Failed     Valid  run-001  进入
```

### 6.2 状态展示
状态应区分：
- Running
- Resumable
- Completed
- Failed
- Invalid
- Missing Workflow

Invalid / Missing 不应只显示红色标签，还需要展示原因摘要。

### 6.3 requirement 摘要
requirement 摘要来自 task authoring 内容。

展示规则：
- 列表中展示 1 行摘要。
- 过长内容截断。
- 单击任务行打开 Task Preview Sheet，抽屉上方直接展示完整需求。
- 完整需求位于独立边框框体中，长内容只在框体内部滚动，不改变抽屉 header 和底部按钮位置。
- 完整需求框右上角提供复制 icon，用于一键复制完整需求。

### 6.4 宽度与刷新规则
任务列表是首页主视觉，Task Preview 以按需右侧抽屉承载：
- 表格列使用固定比例布局，不用内容宽度撑破主内容区。
- 低优先级字段可缩写，例如 `Workflow Valid` 缩写为 `Workflow`、`Artifacts` 缩写为 `Assets`。
- 单元格内容超长时在本列内省略，不触发页面横向滚动。
- Task Preview Sheet 覆盖在内容区右侧，不挤压任务表格；任务表格可在自身容器内横向滚动，页面级不出现横向滚动。
- 刷新时保留当前列表和抽屉状态；用户手动点击刷新时只在刷新按钮和表格卡片顶部使用低对比度局部反馈，不淡化或重绘整页；后台自动刷新每 10 秒静默更新一次，不触发品牌色进度条、按钮高亮或选中行闪烁；只有首次加载才显示整页骨架屏。

---

## 7. 行为

### 7.1 单击任务
单击任务行：
- 打开右侧 Task Preview Sheet。
- 如果抽屉已打开，单击另一任务行时不关闭抽屉，直接切换抽屉内容。
- 单击抽屉外的非任务行区域、按 Escape 或点击关闭按钮时收回抽屉。
- 不立即进入深层页面，避免误触。

### 7.2 双击任务 / 点击进入任务
进入任务工作流页。

页面层级变为：

```text
任务列表 > 任务01 > 工作流
```

### 7.3 点击抽屉底部工作流按钮
抽屉底部固定显示“工作流”按钮：
- 点击后进入对应任务工作流页。
- 抽屉内不提供“查看产物”按钮，避免轻量预览承担资源详情职责。

---

## 8. Task Preview Sheet 布局约束

Task Preview Sheet 是任务列表的轻量详情抽屉，不承担完整任务详情页职责。

布局规则：
- Task Preview 使用 shadcn/ui Sheet 从右侧滑出，默认不自动打开。
- Sheet 使用非模态交互，不用遮罩阻塞任务列表；抽屉打开时仍可单击其他任务行切换内容。
- Sheet header 固定在顶部，完整 requirement 框位于上方内容区，底部固定展示“工作流”按钮。
- requirement 区与其他页面的完整需求抽屉保持同一套白底单框样式：标题行在上、正文只保留一个边框内容框，长文本在框内滚动，不再额外套一层彩色外框。
- requirement 框右上角提供复制 icon，一键复制完整需求文本。
- 抽屉底部“工作流”使用强调色主按钮；抽屉内不展示执行统计，也不展示“查看产物”按钮。
- requirement 文本、task title 和按钮文案都必须在抽屉内换行或截断，不允许撑破容器。
- 页面级不产生横向滚动；任务表格可在自身容器内横向滚动。

验收规则：
- 初始进入任务列表不显示 Task Preview Sheet。
- `document.documentElement.scrollWidth <= document.documentElement.clientWidth`。
- Task Preview Sheet 不展示“执行统计”和“查看产物”。
- 完整需求在框体内可滚动，复制 icon 可复制完整需求。
- 单击任务行打开抽屉，单击另一任务行直接切换内容，单击非任务区域收回抽屉。
- 列表双击进入、表格工具条筛选、关键字搜索、排序、分页和刷新状态保持可用。

---

## 9. 空状态
当没有任务时，展示：
- 当前 workspace 尚无任务
- 创建任务
- 查看示例任务

不显示 command 提示；不展示独立“导入 requirements”入口。

---

## 10. Tauri 2.x MVP 对应实现

MVP 中任务列表由 Tauri command `get_task_list` 提供 view model，前端页面位于 `web/src/pages/TaskListPage.tsx`。

当前实现规则：
- 页面级不展示 summary cards；canonical task/run 派生状态只作为表格工具条筛选条件和列表字段使用。
- 列表行展示 task title/id、requirement preview、display status、workflow 状态和 latest run。
- 单击任务行打开右侧 Task Preview Sheet；抽屉打开时单击另一任务行直接切换内容；双击任务行或点击“进入任务”进入任务工作流页。
- Task Preview Sheet 展示 task id、状态和完整 requirement；完整 requirement 位于可滚动框体内，右上角提供复制 icon，底部固定展示“工作流”按钮，不展示执行统计或产物入口。
- 表格列使用固定比例并在单元格内截断，Task Preview Sheet 不挤压任务列表，避免页面级横向溢出。
- 刷新时保留当前数据，只在刷新按钮和表格卡片顶部叠加局部进度反馈；首次加载使用骨架屏。
- 任务列表页只保留“创建任务”；创建任务 Sheet 内支持导入 `.txt` / `.md` requirement、填写标题/描述、基于 Agent 管理页 agent 创建工作流，Sheet 标题栏右侧提供“保存任务”作为提交入口。
- 任务列表页禁用浏览器/WebView 默认右键菜单，避免用户误以为系统菜单是应用功能。
- 2026-05-03 起页面使用 Tailwind CSS v4 + shadcn/ui Card、Table、Tabs、Button、Skeleton 等现成组件重构；单击选择、双击进入、刷新保留数据和右侧 Task Preview 行为不变。
- 2026-05-05 起任务列表 All / Running / Completed Tabs 是真实筛选控件；表头支持排序；表格底部提供页大小、上一页、下一页和当前范围。
- 2026-05-20 起任务列表默认按 task ID 降序展示，新创建或最新编号任务优先出现在列表顶部；切换到其他列后再次切回 ID 列时也默认先回到降序。
- 2026-05-05 起任务表使用一个统一横向滚动容器和固定 `min-width`，按最大行宽展示各列，避免部分卡片或区域出现不一致滑动条。
- 2026-05-05 验收修正：分页开启后任务列表优先保证表格可读宽度；中小桌面宽度下右侧 Task Preview 下移为全宽卡片，summary cards 使用响应式列数，避免内容被固定右栏挤压截断。
- 2026-05-05 起 summary card 后端提供稳定 key，前端用 i18n 翻译标题；中文模式除技术词外均显示中文，英文模式均显示英文。
- 2026-05-06 起任务列表首页做视觉层级收敛：summary card 改为中性表面 + 小面积状态强调，主内容间距收紧，Task Preview 改为 header 固定、正文内部滚动的安全结构；执行统计在窄栏内单列展示，避免贴边或超出卡片。
- 2026-05-07 起任务列表首页移除页面级 summary cards 和 ModuleBar 状态 tabs，全部任务 / 运行中 / 已完成、可恢复 / 失败 / 配置异常与关键字搜索统一进入表格工具条，首页保持工作台而非数据看板定位。
- 2026-05-09 起 Task Preview 不再使用截断预览 + 完整需求二级视图；抽屉上方直接展示完整需求框，长文本框内滚动并提供复制 icon，底部固定保留“工作流”按钮，移除执行统计和查看产物入口。
- 2026-05-08 起任务列表使用统一 Page Header：面包屑、标题、副标题和页面操作在同一顶部表面中呈现；表格筛选、搜索与分页仍保留在列表卡片内部工具栏。
- 2026-05-08 验收修正：任务列表移除 `资源` 列，不再在主表格展示 `Axx / Pxx` 聚合计数；资源统计仅保留在 Task Preview 等按需上下文中，避免首页列表继续承载低价值字段。
- 2026-05-06 起 Task Preview 从固定右栏改为 shadcn/ui Sheet 右侧抽屉：初始不打开，单击任务行滑出，单击其他任务行直接切换内容，单击非任务区域、Escape 或关闭按钮收回。

---

## 11. 一句话总结

> 任务列表页负责让用户从 requirement 与运行状态出发选择 task，不展示完整 DAG；DAG 从任务工作流页开始出现。
