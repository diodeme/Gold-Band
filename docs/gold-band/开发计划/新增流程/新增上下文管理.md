现在我们处理下侧边栏的上下文管理tab（现有的“知识库”，更名叫上下文管理）
上下文管理中顶部也是一个tab选择页，提供多类上下文管理，本次我们仅支持profile管理（做i18n,中文叫角色管理，英文叫profile管理（？或其他的））
profile管理的就是工作流中的profile
profile支持用户级与项目级：用户级存储在 `~/.gold-band/context/profiles/<name>-<id>.md`，项目级存储在 `~/.gold-band/projects/<project-id>/context/profiles/<name>-<id>.md`。
用户级在项目安装时默认加载：方案、开发、审查、测试、验收、清理等角色，profile在文件管理系统中以md格式存储，md中开头是---块，声明名称、摘要，下面是具体的内容。名称、摘要用于展示和检索，内容用于在节点运行时提供给AI。默认 profile 使用当前生成的分布式 ID，不再保留旧固定 profile id 的迁移兼容逻辑。你先实现默认文件的加载，文件中保留顶部信息块，下面的具体内容我后面再补充。
profile管理中，可以搜索、新增、修改、查看profile，profile在页面中可以是一个卡片，做分页，卡片中包含名称、id、摘要、创建时间、更新时间，查看时显示完整内容，内容需要做markdown格式渲染。新增时要求用户填写名称、摘要、完整内容，id由系统分配为分布式唯一 id；创建时间和更新时间使用系统本地时区的 `YYYY-MM-DD HH:MM:SS`。
工作流定义中，用id做profile的引用，工作流配置面板的profile项做成下拉栏，支持关键字（名称、id、摘要、描述等）索引，下拉栏中每一项展示名称、id、摘要（截取、悬浮展示全文）、创建时间、更新时间，下拉栏默认加载用户目录和当前工作空间的所有profile，选中后下拉栏的当前栏显示名称，名称后跟着一个info图标，悬浮显示名称、id、摘要、创建时间、更新时间。工作流配置面板profile标题，后面跟一个info，悬浮提示，profile可在上下文管理中进行管理，跟一个超链接，点击跳转到上下文管理-->profile管理标签页
