# 系统通知干预弹窗

> 完整设计与校准方案见 `.claude/design/system-notification-intervention-reimpl-plan.md`。本文档为开发计划侧的实现记录，保持与代码同步。

## 1. 背景与边界

- 编排器在 4 类暂停场景（人工确认 / 权限请求 / 执行错误 / 进程中断）下需要主动提醒用户。桌面端 EXE 主路径不保留 terminal 心智，因此通过 **OS 系统通知（Windows Toast）** 单一表面提醒。
- **只涉及弹窗功能本身**，不改动主干编排器控制流、ACP 命令逻辑、session builder、PermissionRequestCard 数据源。
- 弹窗生命周期：**点掉即消失**，无 resolved 闭环。点掉后清 dedup key，允许同节点再次弹出。

## 2. 数据结构（领域划分）

| 结构 | 所属领域 | 位置 |
| --- | --- | --- |
| `InterventionNotification` | 核心库（与 Tauri 无关，可单测） | `src/app/notification.rs` |
| `InterventionType` / `PauseReason→文案映射` | 核心库 | `src/app/notification.rs` |
| `NotificationDedup`（去重 + 软上限淘汰） | 核心库 | `src/app/notification.rs` |
| `App.intervention_notifier` 回调字段 | 核心库 App | `src/app/mod.rs` |
| OS Toast / action 编解码 / AUMID 注册 | 桌面端桥接 | `src-tauri/src/notifications.rs` |
| `DesktopState.notification_dedup` | 桌面端状态 | `src-tauri/src/state.rs` |
| 前端 VM / 订阅 / hook / 卡片 | 前端 | `web/src/...` |

### dedup key

`run:round:node:attempt:reason`（reason 为 kebab-case，如 `permission-requested`）。**不含 request_id**——同一节点同一轮同一原因只弹一次。

### PauseReason → 文案映射（集中在 `InterventionNotification::new`）

| PauseReason | title | InterventionType |
| --- | --- | --- |
| `WaitingForUserInput` | 人工确认 | `manualCheck` |
| `PermissionRequested` | 权限请求 | `permissionRequest` |
| `ErrorBlocked` | 执行错误 | `errorBlocked` |
| `ProcessInterrupted` | 进程中断 | `errorBlocked` |

`body = {task_label} · {node_label} {body_template}`，`task_title` 缺失时回退 task_id。

## 3. 生命周期与去重

- **点掉即消失**：OS Toast「忽略」/「查看详情」按钮点击时，后端 `handle_toast_action` 清 dedup key（允许同节点再次弹出）。
- **查看详情**：后端 emit 导航事件 → 前端 deep-link 到目标节点（前端订阅 `gold-band://intervention-navigate`）。
- **无 resolved 事件、无 `resolve_intervention`**——弹窗消失是 OS Toast 动作，无前端卡片。
- `clear_run`：run 终态或通知治理需要时可批量清该 run 的所有 key（内存治理，防常驻 EXE 泄漏）；软上限 5000 FIFO 淘汰兜底。

## 4. 接入路径

### 路径 A：编排器 4 个暂停入口（`src/app/orchestrator.rs`）

抽 `emit_intervention_notification` helper，在 4 个分散暂停入口各追加一行调用，**不改控制流**：

| 入口 | 场景 |
| --- | --- |
| A | 节点执行失败暂停（`failed_node`） |
| B | 节点暂停 |
| C | 人工 check 暂停 |
| D | `apply_control_decision` PauseRun |

reason 直接读 `run.pause_reason`。`:6252` 的 Paused 是恢复路径，**不挂接**。

### 路径 B：ACP 旁路监听 `permissionRequest`（`src-tauri/src/commands.rs`）

`acp_live_update_emitter` 中检测 `kind=="permissionRequest" && status=="pending"`，构造 `InterventionNotification(PermissionRequested)` 发送。使用通用文案（node_label=node_id，task_title=None），不改主干 session 逻辑。

## 5. OS 系统通知（Windows Toast，`src-tauri/src/notifications.rs`）

- `tauri-winrt-notification` 0.7：`Toast::new(AUMID).title().text1().duration(Long).scenario(Reminder).add_button("查看详情", view).add_button("忽略", dismiss).on_activated(...).icon(...).show()`。
- AUMID = `local.gold-band.desktop`（与 tauri.conf identifier 一致）。`Once` 注册 AUMID + 校验/重建开始菜单 .lnk（TargetPath 不符则重建）。
- action 编码：`view:` / `dismiss:` + base64(url_safe_no_pad, json)。
- `on_activated`：view → `clear_key` + emit `gold-band://intervention-navigate`；dismiss → `clear_key`；None → debug。
- **图标**：`include_bytes!("../icons/icon.png")` 编译期嵌入码灵图标字节；运行时 `Once` 幂等写入 `app_local_data_dir/maling-toast-icon.png`，`toast.icon(path, Square, "码灵")` 绑定 Toast 左上角。不走运行时 `BaseDirectory::Resource` 解析（dev/prod 落空会导致图标回退默认）。
- macOS/Linux 走 `notify-rust`（`native-notification` feature，非默认平台）。

## 6. 事件契约

| 事件 | payload | 方向 |
| --- | --- | --- |
| `gold-band://intervention-navigate` | `{taskId, runId, roundId, nodeId, attemptId, dedupKey}` | OS Toast「查看详情」→ 前端导航 |

> 应用内右上角弹窗已移除：不再有 `gold-band://intervention-required` 入队事件，不再有 `dismiss_intervention` / `dismiss_intervention_by_fields` 命令。去重清理统一由后端 `handle_toast_action` 在 Toast 按钮点击时完成。

## 7. 前端集成（`web/src/...`）

- `types.ts`：`InterventionNavigateEventVm`（仅导航事件 payload）。
- `api/client.ts` + `desktop.ts` + `browser.ts` + `api.ts`：`subscribeInterventionNavigate`。
- `lib/use-intervention-notifications.ts`：稳定订阅 `gold-band://intervention-navigate`（`ref` + `useEffect([])`，无重订阅），触发 `onNavigate` deep-link。无队列、无 dismiss、无孤儿兜底。
- `App.tsx`：按 uiMode 导航——
  - **工作台**：`round-detail` + `roundSelection.node`。
  - **会话**：定位到 `conversation-run`，在 `sessionTree` 内按 `(roundId, nodeId, attemptId)` 匹配叶子（含 outer 字段）后 `switchConversationSession`。projectId 由 sidebar `tasksByWorkspace` 反查 taskId 得到。

> 应用内右上角堆叠卡片组件 `components/InterventionNotificationQueue.tsx` 已删除。
